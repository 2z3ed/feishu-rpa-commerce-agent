# Feishu RPA Commerce Agent

一个基于飞书自然语言入口的电商后台自动化系统。

系统通过自然语言接收业务指令，完成任务创建、异步执行、业务服务调用、价格监控、采集诊断、决策建议与结果回传。

---

## 一、项目定位

本项目不是普通聊天机器人，也不是单点脚本工具。

它的目标是构建一个可执行、可追踪、可解释的电商后台自动化系统。

用户在飞书中发送自然语言命令后，系统会完成：

- 接收消息
- 幂等处理
- 创建任务
- 异步执行
- 调用业务服务 / RPA
- 返回结果
- 记录执行过程
- 提供后续查询与诊断

核心目标：

> 用自然语言驱动电商后台操作，并保证可控、可追踪、可解释。

## 五、系统特点

### 1. 可执行

系统不是只返回文本，而是能够真正触发后端任务，执行价格刷新、状态更新、URL 替换、重新采集等操作。

### 2. 可追踪

每次执行都会生成任务记录、步骤记录、刷新批次和执行明细。

### 3. 可解释

系统会输出价格来源、采集状态、可信度、异常原因和处理建议。

### 4. 可治理

当数据源质量较差时，系统支持发现问题、替换 URL、重新采集、更新诊断结果。

### 5. 可扩展

当前能力集中在价格监控，但架构可扩展到库存、商品、客服、订单等更多电商后台场景。

## 八、A / B 项目边界

本项目作为 A 项目，主要负责：

```text
飞书入口
自然语言解析
任务编排
结果回传
卡片 / 文本展示
用户交互
```

配套 B 项目负责：

```text
监控对象
价格刷新
价格采集
价格历史
刷新批次
采集诊断
决策建议
```

边界原则：

```text
A 负责入口与交互
B 负责业务能力
A 不吞 B
B 不接管飞书入口
```
