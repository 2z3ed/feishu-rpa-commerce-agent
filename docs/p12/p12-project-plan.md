# P12-E 开发主线文档

## 阶段名称

P12-E：飞书卡片交互层收口与演示稳定版

## 一、阶段背景

P12-A / B / C / D 已经完成了飞书卡片交互层的核心能力：

- P12-A：搜索结果卡片展示
- P12-B：候选卡片“加入监控”按钮
- P12-C：监控对象管理卡片，支持暂停 / 恢复
- P12-D：监控对象超过 5 个时支持“查看更多”

现在 P12 不应该继续马上堆新按钮，而应该先做阶段收口。

P12-E 的目标不是新增功能，而是让当前卡片交互层：

- 可演示
- 可回归
- 可交接
- 边界清楚
- 后移项清楚

## 二、本轮唯一目标

只做：

P12 卡片交互层收口与演示稳定。

具体包括：

1. README 中统一 P12 阶段说明
2. docs/p12 中形成 P12-E 收口口径
3. boss demo SOP 串成完整演示路径
4. acceptance checklist 固化回归项
5. 固化 P12 回归测试命令
6. 统一卡片失败 / fallback / 旧卡片失效等老板可读文案
7. 明确后移项：删除、批量、搜索过滤、排序

## 三、本轮禁止做

禁止：

- 不做删除按钮
- 不做批量管理
- 不做搜索过滤
- 不做排序
- 不做 PostgreSQL
- 不新增业务动作
- 不新增复杂卡片状态同步
- 不重写 P12-A / P12-B / P12-C / P12-D
- 不破坏现有卡片交互链路

## 四、本轮允许做

允许：

1. 更新 README P12 阶段说明
2. 更新 docs/p12 四份文档为 P12-E 收口口径
3. 新增 P12 收口文档，如 docs/p12/p12-closure-summary.md
4. 新增 P12 回归脚本，如 scripts/p12_regression_check.sh
5. 统一错误提示文案
6. 补少量测试，验证文案和 fallback 不退化
7. 清理临时日志，但不能删除必要诊断日志

## 五、收口演示路径

P12-E 后，老板演示路径应固定为：

```text
搜索商品：重力毯
```

预期：

- 返回候选结果卡片
- 每条候选有“加入监控”

然后点击：

```text
加入监控
```

预期：

- 成功纳管
- 结果优先回复当前群聊

然后发送：

```text
看看当前监控对象
```

预期：

- 返回管理卡片
- 展示 active / inactive
- 有暂停 / 恢复按钮
- 超过 5 个时有“查看更多”

然后点击：

```text
查看更多
```

预期：

- 返回下一页对象
- 暂停 / 恢复按钮仍可用

然后点击：

```text
暂停监控 / 恢复监控
```

预期：

- 操作成功
- 补发老板可读文本

## 六、后移项

P12-E 明确后移：

- P12-F：删除监控对象确认版
- P12-G：批量管理版

- 已固化 P12 回归测试命令
