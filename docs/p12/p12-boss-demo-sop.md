# P12-D 老板演示 SOP

## 一、演示目标

验证 P12-D 是否解决了 P12-C 的后移项：

当监控对象超过 5 个时，老板可以通过卡片“查看更多”继续查看第 6 条以后的对象。

## 二、演示前提

需要启动：

1. B 项目：Ecom-Watch-Agent-Agent
2. A 项目 API
3. A 项目 worker
4. 飞书长连接 listener
5. 飞书机器人在目标群聊可用

B 默认地址：

```text
http://127.0.0.1:8005
```

## 三、演示前准备

需要确保当前监控对象数量超过 5 个。

可先发送：

```text
看看当前监控对象
```

如果不足 6 个，可以通过搜索和加入监控补足。

## 四、演示步骤

### 步骤 1：查看第一页

飞书发送：

```text
看看当前监控对象
```

预期：

- 返回监控对象管理卡片
- 展示前 5 条对象
- 如果总数超过 5，底部有“查看更多”

通过标准：

- 是卡片，不是纯文本
- 最多展示 5 条
- 有“查看更多”
- 每条对象字段完整
- 暂停 / 恢复按钮仍可见

### 步骤 2：点击查看更多

点击：

```text
查看更多
```

预期：

- 返回下一页管理卡片
- 展示第 6 条及后续对象
- 仍展示名称、ID、状态、URL
- 仍有暂停 / 恢复按钮

通过标准：

- 能看到第 6 条以后的对象
- 不重复只显示前 5 条
- 不吐堆栈
- 不静默失败

### 步骤 3：验证下一页按钮

如果还有更多对象：

- 继续显示“查看更多”

如果没有更多对象：

- 不显示“查看更多”
- 或显示“没有更多监控对象”

### 步骤 4：回归暂停 / 恢复

在第 2 页点击一个 active 对象的“暂停监控”。

预期：

```text
已暂停监控
```

在 inactive 对象点击“恢复监控”。

预期：

```text
已恢复监控
```

### 步骤 5：回归 P12-A / P12-B

发送：

```text
搜索商品：重力毯
```

预期：

- 仍返回候选卡片
- 候选卡片“加入监控”仍可用

再发送：

```text
加入监控第 1 个
```

预期：

- 文本纳管仍可用

## 五、失败样本

可选：

- 点击旧卡片里的查看更多
- 构造非法 page
- B 服务关闭后点击查看更多

预期：

- 返回老板可读错误
- 不吐堆栈
- 不静默失败
