# P12-E 老板演示 SOP

## 一、演示目标

P12-E 的演示目标是完整展示 P12 卡片交互层闭环。

本轮不是验证新功能，而是验证 P12-A / B / C / D 能否稳定串成一条老板可理解的演示路径。

## 二、演示前提

需要启动：

1. B 项目：Ecom-Watch-Agent-Agent
2. A 项目 API
3. A 项目 worker
4. 飞书长连接 listener
5. 飞书机器人在目标群聊可用

## 三、完整演示路径

### 步骤 1：搜索商品

发送：

```text
搜索商品：重力毯
```

预期：

- 返回候选结果卡片
- 卡片展示前 5 条候选
- 每条候选有“加入监控”

### 步骤 2：点击加入监控

点击候选卡片中的：

```text
加入监控
```

预期：

- 成功加入监控
- 结果回复到当前群聊
- 文本包含对象ID和状态

### 步骤 3：查看监控对象

发送：

```text
看看当前监控对象
```

预期：

- 返回管理卡片
- 展示对象名称、ID、状态、URL
- active 显示暂停
- inactive 显示恢复

### 步骤 4：查看更多

如果对象超过 5 个，点击：

```text
查看更多
```

预期：

- 返回下一页
- 能看到第 6 条以后的对象
- 下一页仍有暂停 / 恢复按钮

### 步骤 5：暂停 / 恢复

点击：

```text
暂停监控
```

或：

```text
恢复监控
```

预期：

- 操作成功
- 补发老板可读文本
- 不吐堆栈

### 步骤 6：文本链路回归

发送：

```text
加入监控第 1 个
```

预期：

- 文本方式仍可用

## 四、失败场景演示

可选演示：

1. B 服务关闭时搜索商品
2. 点击旧卡片按钮
3. 点击没有更多页的查看更多
4. payload 缺失或非法

预期：

- 返回老板可读错误
- 不吐堆栈
- 不静默失败

## 五、演示通过标准

P12-E 演示通过条件：

- 搜索卡片通过
- 加入监控按钮通过
- 管理卡片通过
- 查看更多通过
- 暂停 / 恢复通过
- 文本链路回归通过
- 失败提示可读
