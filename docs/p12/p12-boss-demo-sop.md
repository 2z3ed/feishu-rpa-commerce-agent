# P12-C 老板演示 SOP

## 一、演示目标

验证“看看当前监控对象”是否从纯文本列表升级为飞书管理卡片。

P12-C 不是新增 discovery 能力。  
P12-C 也不是继续改候选卡片。

本轮演示目标：

老板可以用卡片查看当前监控对象，并在能力允许时点击暂停 / 恢复按钮。

## 二、演示前提

需要启动：

1. B 项目：Ecom-Watch-Agent-Agent
2. A 项目 API
3. A 项目 worker
4. 飞书长连接 listener
5. 飞书机器人在目标群聊可用

B 默认地址：

```text
http://127.0.0.1:8005
三、演示前准备

建议先确保已有监控对象：

搜索商品：亚马逊 U型抱枕

点击候选卡片中的“加入监控”。

或者使用文本：

加入监控第 1 个

然后确认：

看看当前监控对象
四、P12-C 演示步骤
步骤 1：查看当前监控对象

飞书发送：

看看当前监控对象

预期：

先收到“已接收任务”
后收到监控对象管理卡片
卡片展示当前监控对象总数
卡片展示前 5 条对象

通过标准：

不是纯文本列表
是飞书卡片
对象名称可读
状态可读
URL 可读
ID 可读
步骤 2：检查按钮展示

观察卡片：

active 对象应出现“暂停监控”
inactive 对象应出现“恢复监控”
不应出现“删除”按钮

通过标准：

按钮和状态匹配
没有删除按钮
没有分页按钮
步骤 3：点击暂停或恢复

如果 B 服务已有 pause / resume 能力：

点击一个 active 对象的“暂停监控”。

预期：

已暂停监控。
- 对象ID：7
- 状态：inactive

再点击 inactive 对象的“恢复监控”。

预期：

已恢复监控。
- 对象ID：7
- 状态：active

如果 B 服务没有 pause / resume 能力：

本步骤不作为失败，但必须记录为后移项。

步骤 4：回归 P12-A

飞书发送：

搜索商品：重力毯

预期：

仍能返回候选结果卡片。

步骤 5：回归 P12-B

点击候选卡片中的“加入监控”。

预期：

仍能加入监控，并补发“已加入监控”文本。

步骤 6：回归文本方式

发送：

加入监控第 2 个

预期：

仍能通过文本方式加入监控。

五、失败样本

可选：

B 服务关闭后发送“看看当前监控对象”

点击旧卡片按钮

预期：

返回老板可读错误
