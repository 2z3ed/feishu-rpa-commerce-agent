# P12-F 老板演示 SOP

## 一、演示目标

P12-F 的演示目标是验证：

监控对象可以安全删除，但删除必须经过二次确认。

本轮不是批量管理，不是搜索过滤，也不是排序。

## 二、演示前提

需要启动：

1. B 项目：Ecom-Watch-Agent-Agent
2. A 项目 API
3. A 项目 worker
4. 飞书长连接 listener
5. 飞书机器人在目标群聊可用

## 三、演示前准备

确保当前至少有 1 个监控对象。

可先执行：

```text
搜索商品：重力毯
```

然后点击：

```text
加入监控
```

确认对象存在：

```text
看看当前监控对象
```

## 四、演示步骤

### 步骤 1：打开监控对象管理卡片

发送：

```text
看看当前监控对象
```

预期：

- 返回管理卡片
- 对象展示名称、ID、状态、URL
- 有暂停 / 恢复按钮
- 有删除监控入口

### 步骤 2：点击删除监控

点击某个对象的：

```text
删除监控
```

预期：

- 不能直接删除
- 系统返回删除确认卡片

确认卡片必须展示：

- 对象名称
- 对象ID
- URL
- 当前状态
- 风险提示

### 步骤 3：点击取消

点击：

```text
取消
```

预期：

- 不删除对象
- 返回“已取消删除”
- 再发送“看看当前监控对象”，对象仍存在

### 步骤 4：再次点击删除并确认

再次点击：

```text
删除监控
```

然后点击：

```text
确认删除
```

预期：

- 系统调用删除能力
- 返回“已删除监控对象”
- 再发送“看看当前监控对象”，该对象不再出现

### 步骤 5：回归 P12-C / P12-D

验证：

- 暂停 / 恢复仍可用
- 查看更多仍可用
- 文本加入监控仍可用

## 五、失败场景

可选：

1. 点击旧确认卡片
2. 删除不存在的 target_id
3. B 服务关闭后确认删除
4. payload 缺失 target_id

预期：

- 返回老板可读错误
- 不吐堆栈
- 不静默失败

## 七、本轮不演示

不演示：

- 批量删除
- 批量暂停
- 搜索过滤
- 排序
- PostgreSQL
- 权限审批
