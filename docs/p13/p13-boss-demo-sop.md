# P13-D 老板演示 SOP

## 一、演示目标

P13-D 的演示目标是证明：

手动刷新监控价格后，系统会生成一个可追踪的刷新批次 run_id，并且可以通过飞书查询本次刷新详情。

本轮不是定时任务，不是阈值提醒，也不是调度系统。

本轮只演示：

```text
刷新价格
→ 生成刷新批次
→ 查询刷新批次详情
```

## 二、演示前提

需要启动：

A 项目：

```text
feishu-rpa-commerce-agent
```

B 项目：

```text
Ecom-Watch-Agent-Agent
```

需要确保：

- B 服务运行在 http://127.0.0.1:8005
- A API / worker / 飞书长连接 listener 已启动
- 已有 active 监控对象
- P13-C “刷新监控价格”可用

## 三、演示步骤

### 步骤 1：刷新监控价格

飞书发送：

```text
刷新监控价格
```

预期：

返回中包含：

```text
刷新批次：PRR-20260425-xxxx
```

同时保留 P13-C 变化摘要：

```text
本轮价格变化：3 个
...
```

### 步骤 2：查询刷新批次

飞书发送：

```text
查看刷新结果 PRR-20260425-xxxx
```

或：

```text
查看价格刷新批次 PRR-20260425-xxxx
```

预期：

返回：

```text
价格刷新结果：PRR-20260425-xxxx
状态：succeeded
总对象数：10
成功刷新：10
价格变化：3
失败：0
耗时：1200ms
```

### 步骤 3：查看变化对象

查询结果中应展示变化对象：

```text
变化对象：
1. xxx
   当前价：199
   上次价：209
   变化：下降 10（-4.78%）
```

### 步骤 4：查询不存在 run_id

飞书发送：

```text
查看刷新结果 PRR-NOT-FOUND
```

预期：

- 返回老板可读错误
- 不吐堆栈
- 不静默失败

### 步骤 5：回归 P13-B

飞书发送：

```text
查看价格历史 7
```

预期：

价格历史仍可查询。

### 步骤 6：回归 P12

验证：

- 候选卡片加入监控仍可用
- 管理卡片暂停 / 恢复仍可用
- 查看更多仍可用
- 删除二次确认仍可用

## 四、失败场景

可选：

1. B 服务关闭后查询 run_id
2. run_id 格式非法
3. run_id 不存在
4. run 详情中存在 failed item

预期：

- 返回老板可读错误
- 不吐堆栈
- 不静默失败
