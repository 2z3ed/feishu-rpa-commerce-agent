# P13-C 老板演示 SOP

## 一、演示目标

P13-C 的演示目标是证明：

手动刷新监控价格后，系统会把本轮价格变化以老板可读摘要返回飞书。

本轮不是阈值提醒，不是定时任务，也不是主动推送。

本轮只演示：

```text
刷新价格
→ 识别本轮变化
→ 返回变化摘要
```

## 二、演示前提

需要启动：

A 项目：

```text
feishu-rpa-commerce-agent
```

B 项目：

```text
Ecom-Watch-Agent-Agent
```

需要确保：

- B 服务运行在 http://127.0.0.1:8005
- A API / worker / 飞书长连接 listener 已启动
- 已有 active 监控对象
- P13-A “刷新监控价格”可用
- P13-B “查看价格历史”可用

## 三、演示步骤

### 步骤 1：刷新监控价格

飞书发送：

```text
刷新监控价格
```

预期：

如果有变化，返回：

```text
监控价格已刷新。

本轮价格变化：3 个
1. XXX
   当前价：199
   上次价：209
   变化：下降 10（-4.78%）

未变化：7 个
刷新失败：0 个
```

如果无变化，返回：

```text
监控价格已刷新。
本轮暂无价格变化。
成功刷新：10 个
失败：0 个
```

### 步骤 2：变化对象数量超过 5 条

如果变化对象超过 5 条：

预期：

- 只展示前 5 条
- 显示：

```text
还有 X 个价格变化对象未展示。
```

### 步骤 3：回归价格历史

飞书发送：

```text
查看价格历史 7
```

预期：

- 价格历史仍可查询
- 本轮刷新产生的历史仍可看到

### 步骤 4：回归管理卡片

飞书发送：

```text
看看当前监控对象
```

预期：

- 管理卡片仍展示 current_price / last_price / delta
- 暂停 / 恢复 / 删除 / 查看更多不退化

### 步骤 5：回归 P12

验证：

- 候选卡片加入监控仍可用
- 暂停 / 恢复仍可用
- 删除二次确认仍可用

## 四、失败场景

可选：

1. B 服务关闭后刷新价格
2. 某些对象刷新失败
3. B 返回 items 为空但 changed > 0

预期：

- 返回老板可读错误或降级摘要
- 不吐堆栈
- 不静默失败
