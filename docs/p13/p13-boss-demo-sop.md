# P13-I 老板演示 SOP

## 一、演示目标

P13-I 的演示目标是证明：

系统不只是采集价格，还能判断价格是否可信、页面类型是否合适、价格变化是否异常，并给出一句轻量处理建议。

本轮不是完整决策建议系统，不是主动通知，不是复杂规则引擎。

## 二、演示前提

需要启动：

A 项目：

```text
feishu-rpa-commerce-agent
```

B 项目：

```text
Ecom-Watch-Agent-Agent
```

需要确保：

- B 服务运行在 http://127.0.0.1:8005
- A API / worker / 飞书长连接 listener 已启动
- 已有 success / fallback_mock / mock_price / suspected 样本

## 三、演示步骤

### 步骤 1：刷新价格或重试采集

飞书发送：

```text
刷新监控价格
```

或：

```text
重试价格采集
```

预期：

- 价格刷新正常
- 诊断字段随刷新更新

### 步骤 2：查看管理卡片

飞书发送：

```text
看看当前监控对象
```

预期每个对象展示：

```text
可信度：high / medium / low
页面类型：product_detail / listing_page / article_page
异常状态：normal / suspected
异常原因：-
建议：该价格可作为当前监控参考
```

### 步骤 3：查看价格异常对象

飞书发送：

```text
查看价格异常对象
```

预期：

```text
价格异常对象（共 X 个）：
...
异常原因：
建议：
```

### 步骤 4：查看低可信价格对象

飞书发送：

```text
查看低可信价格对象
```

预期：

```text
低可信价格对象（共 X 个）：
...
可信度：
页面类型：
建议：
```

### 步骤 5：查看价格监控状态

飞书发送：

```text
查看价格监控状态
```

或：

```text
价格监控概览
```

预期：

```text
价格监控状态

监控对象总数：13
高可信价格：3
中可信价格：2
低可信价格：8
异常价格：2
mock/fallback：6

建议：
- 优先复查 2 个异常价格对象
- 处理 8 个低可信对象
- mock/fallback 对象不建议用于价格决策
```

## 四、失败场景

可选：

1. 没有异常对象
2. 没有低可信对象
3. 所有对象都是 unknown
4. B 服务返回旧数据无诊断字段

预期：

- 返回老板可读文本
- 不吐堆栈
- 不静默失败

## 五、回归验证

必须回归：

