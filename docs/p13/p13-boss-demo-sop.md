# P13-A 老板演示 SOP

## 一、演示目标

P13-A 的演示目标是证明：

监控对象已经从“URL 列表”升级为“带价格数据的监控对象”。

本轮不是复杂爬虫演示。

本轮只演示最小价格数据闭环：

```text
刷新价格
→ 保存价格
→ 计算变化
→ 飞书卡片展示
```

## 二、演示前提

需要启动：

A 项目：

```text
feishu-rpa-commerce-agent
```

B 项目：

```text
Ecom-Watch-Agent-Agent
```

需要确保：

- B 服务运行在 http://127.0.0.1:8005
- A API / worker / 飞书长连接 listener 已启动
- 已有至少 1 个监控对象

## 三、演示步骤

### 步骤 1：确认已有监控对象

飞书发送：

```text
看看当前监控对象
```

预期：

- 返回管理卡片
- 每个对象仍展示名称、ID、状态、URL
- 未采集价格的对象显示：

```text
当前价格：暂未采集
```

### 步骤 2：刷新监控价格

飞书发送：

```text
刷新监控价格
```

预期：

返回老板可读汇总：

```text
监控价格已刷新。
- 总对象数：8
- 成功刷新：6
- 价格变化：2
- 失败：0
```

### 步骤 3：再次查看监控对象

飞书发送：

```text
看看当前监控对象
```

预期：

管理卡片展示价格信息：

```text
当前价格：199
上次价格：暂未采集
变化：暂无
最后检测：2026-04-25 17:30
来源：mock_price
```

### 步骤 4：第二次刷新价格

再次发送：

```text
刷新监控价格
```

预期：

- current_price 更新
- old current_price 进入 last_price
- 计算 price_delta
- 计算 price_delta_percent

### 步骤 5：再次查看价格变化

飞书发送：

```text
看看当前监控对象
```

预期：

卡片中展示：

```text
当前价格：199
上次价格：209
变化：下降 10（-4.78%）
最后检测：2026-04-25 17:30
```

## 四、P12 回归演示

必须确认：

- 候选卡片加入监控仍可用
- 监控对象暂停 / 恢复仍可用
- 查看更多仍可用
- 删除二次确认仍可用

## 五、失败场景

可选演示：

1. B 服务关闭后刷新价格
2. 某个对象价格未知
3. price_source 不可用

预期：

- 返回老板可读错误
- 不吐堆栈
- 未采集对象显示“暂未采集”
