# P13-H 老板演示 SOP

## 一、演示目标

P13-H 的演示目标是证明：

系统不仅能发现价格采集失败，还能对失败 / fallback / mock 对象做一次人工触发的轻量重试。

本轮不是自动重试队列，不是定时重试，不是告警系统。

## 二、演示前提

需要启动：

A 项目：

```text
feishu-rpa-commerce-agent
```

B 项目：

```text
Ecom-Watch-Agent-Agent
```

需要确保：

- B 服务运行在 http://127.0.0.1:8005
- A API / worker / 飞书长连接 listener 已启动
- 至少有一个 fallback_mock 或 mock_price 对象
- 至少有一个可成功 html_extract_preview 的样本

## 三、演示步骤

### 步骤 1：查看失败 / mock 对象

飞书发送：

```text
查看价格采集失败
```

或：

```text
查看mock价格对象
```

预期：

能看到需要重试的对象列表。

### 步骤 2：批量重试

飞书发送：

```text
重试价格采集
```

预期：

返回：

```text
价格采集重试完成。

重试对象：5 个
成功转真实价格：2 个
仍失败：3 个
```

### 步骤 3：单对象重试

飞书发送：

```text
重试对象ID 6 价格采集
```

预期：

成功时返回：

```text
价格采集重试成功。
对象ID：6
当前价格：1280.0
来源：html_extract_preview
```

失败时返回：

```text
价格采集重试后仍未成功。
对象ID：9
状态：fallback_mock
原因：timeout
来源：mock_price
```

### 步骤 4：查看管理卡片

飞书发送：

```text
看看当前监控对象
```

预期：

重试成功对象：

```text
采集状态：success
来源：html_extract_preview
```

仍失败对象：

```text
采集状态：fallback_mock
采集原因：timeout
来源：mock_price
```

### 步骤 5：回归 P13-G

飞书发送：

```text
查看真实价格对象
查看mock价格对象
```

预期：

对象分类随重试结果更新。

## 四、失败场景

可选：

1. 单对象不存在
2. 对象 URL 不可访问
3. HTML 没有价格
4. 批量无可重试对象

预期：

- 返回老板可读错误
- 不吐堆栈
- 不静默失败

## 五、回归验证

必须回归：

```text
刷新监控价格
查看价格采集失败
查看真实价格对象
查看价格历史 7
查看刷新结果 PRR-...
看看当前监控对象
```

P12：

- 加入监控
- 暂停 / 恢复
- 删除确认
