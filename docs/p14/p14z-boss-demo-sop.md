# P14-Z 老板演示 SOP：P14 LLM 智能增强层总收口

## 一、演示目标

本 SOP 用于说明 P14 阶段最终交付内容。

P14 不是让 LLM 自动操作后台，而是在原有规则闭环基础上加入智能增强层，让系统具备：

- 听懂自然语言
- 总结监控情况
- 解释异常原因
- 生成处理计划

但系统仍然不自动执行高风险动作。

## 二、P14 四个能力

### 1. P14-A：听懂

用户可以用更自然的话描述需求。

系统在规则未命中时，通过 LLM fallback 解析 intent 和 slots。

例如：

- 帮我看看哪些商品不太对
- 失败的那些再跑一遍
- 帮我把 SKU A001 的价格改成 99

改价类动作仍然必须进入确认链路。

### 2. P14-B：总结

用户可以问：

- 总结一下当前价格监控情况
- 帮我看一下现在监控整体怎么样
- 当前有哪些商品需要重点处理

系统能返回老板可读的监控总结。

### 3. P14-C：解释

用户可以问：

- 为什么这些商品价格不准
- 解释一下低可信对象的问题
- mock_price 是什么意思
- 为什么这些商品需要人工处理

系统能把诊断字段翻译成业务解释。

### 4. P14-D：计划

用户可以问：

- 这些异常商品下一步怎么处理
- 帮我安排一下处理顺序
- 低可信对象接下来怎么处理
- 哪些先重试，哪些先换 URL

系统能生成下一步处理计划。

## 三、安全边界

P14 不会自动执行：

- 自动刷新
- 自动重试
- 自动替换 URL
- 自动删除对象
- 自动改价
- 自动调用 RPA
- 自动发送告警

所有高风险动作仍然要走原规则和确认链路。

## 四、演示建议

演示顺序建议：

1. 先演示 P14-B 总结
2. 再演示 P14-C 解释
3. 再演示 P14-D 计划
4. 最后说明 P14-A 的 fallback 是底层理解能力

## 五、收口结论

P14 完成后，系统从“规则命令机器人”升级为“规则可控 + LLM 辅助理解/总结/解释/计划”的电商 RPA 智能入口。
