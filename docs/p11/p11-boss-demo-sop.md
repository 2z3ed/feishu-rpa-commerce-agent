# P11-D 老板演示 SOP

## 一、演示目标

让老板在飞书里直接体验：

先查看当前监控对象
→ 再按编号暂停 / 恢复 / 删除一个对象
→ 再查看当前监控对象确认状态变化

## 二、演示固定样本

建议固定样本：

第一步飞书发送：
- 看看当前监控对象

第二步飞书发送：
- 暂停监控第 2 个

第三步飞书发送：
- 看看当前监控对象

第四步飞书发送：
- 恢复监控第 2 个

第五步飞书发送：
- 看看当前监控对象

固定失败样本（推荐）：
- 删除监控第 99 个

## 三、演示前准备

1. 启动 A 主系统
2. 启动 A 的飞书 listener
3. 启动 B 服务
4. 确认 B 运行在 127.0.0.1:8005
5. 确认 B 的以下接口可调用：
   - POST /internal/monitor/{id}/pause
   - POST /internal/monitor/{id}/resume
   - DELETE /internal/monitor/{id}
6. 确认 A → B HTTP 调用可用
7. 确认 Envelope 解包正常

## 四、演示步骤

### 第一步：查看当前监控对象
发送：
看看当前监控对象

预期飞书返回：
- 当前监控对象列表
- 至少若干条对象
- 带序号

固定真实样本（冻结）：
- task_id：`TASK-20260424-35D9F2`
- 回复原文：

  当前监控对象（共 5 个）：

  * 1. Mock Phone X（inactive，ID=1）
  * 2. Mock Headphone Pro（active，ID=2）
  * 3. Mock Keyboard Mini（inactive，ID=3）
  * 4. abc（active，ID=4）
  * 5. 蓝牙耳机 | 香港蘇寧 SUNING（active，ID=5）

### 第二步：暂停其中一个对象
发送：
暂停监控第 2 个

预期飞书返回：
- 已暂停监控
- 对象名称 / 对象信息（若有）

固定真实样本（冻结）：
- task_id：`TASK-20260424-240919`
- 回复原文：

  已暂停监控。

  * 选择编号：第 2 个
  * 名称：Mock Headphone Pro
  * 对象ID：2

### 第三步：再次查看列表
发送：
看看当前监控对象

预期飞书返回：
- 第 2 个对象状态已变化（例如 inactive / paused）

固定真实样本（冻结）：
- task_id：`TASK-20260424-E20D19`
- 验证点：回复原文中第 2 个对象状态变为 `inactive`

### 第四步：恢复该对象
发送：
恢复监控第 2 个

预期飞书返回：
- 已恢复监控

固定真实样本（冻结）：
- task_id：`TASK-20260424-219943`
- 回复原文：

  已恢复监控。

  * 选择编号：第 2 个
  * 名称：Mock Headphone Pro
  * 对象ID：2

### 第五步：再次查看列表
发送：
看看当前监控对象

预期飞书返回：
- 第 2 个对象状态恢复 active

固定真实样本（冻结）：
- task_id：`TASK-20260424-84BF42`
- 验证点：回复原文中第 2 个对象状态恢复 `active`

## 五、失败样本（至少一个）

固定失败样本（冻结）：
- 飞书原文：`删除监控第 99 个`
- task_id：`TASK-20260424-5C0A5E`
- 回复原文：`操作失败：编号超出范围（当前最多 5 个）`

预期飞书返回：
- 操作失败
- 原因简述
- 不出现 Python 堆栈
- 不出现原始 JSON

## 六、演示成功标准

1. 飞书列表命令能进入 A
2. A 能真实调到 B 的 targets
3. 飞书管理命令能进入 A
4. A 能真实调到 B 的 pause / resume / delete
5. 飞书能看到老板可读结果
6. 再看当前监控对象时能看到状态变化
7. 不破坏 P10 / P11-A / P11-B / P11-C 已收口链路
