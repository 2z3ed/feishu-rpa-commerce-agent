# P11-B 老板演示 SOP

## 一、演示目标

让老板在飞书里直接体验：

发一个搜索词
→ 系统返回候选商品列表
→ 老板先看到“发现能力”已成立

本轮不要求：
- 直接加入监控
- 候选编号选择
- 卡片交互

## 二、演示固定样本

本阶段收口后，固定样本如下：

### 成功样本

- 飞书原文：`搜索商品：蓝牙耳机`
- task_id：`TASK-20260423-8089C1`
- 回复原文：
  - 搜索结果：蓝牙耳机
  - 批次：4
  - 候选（展示前 5 条）：
    1. 藍牙耳機 - Fortress豐澤
    2. 蓝牙耳机 | 香港蘇寧 SUNING
    3. 2026年蓝牙耳机选购指南，高性价比蓝牙耳机推荐（4月更新） - 知乎
    4. 小米耳機| 小米®香港官方商城 - Xiaomi
    5. 蓝牙耳机_百度百科

### 失败样本

- 飞书原文：`搜索商品：`
- task_id：`TASK-20260423-8765A3`
- 回复原文：`搜索失败：请输入搜索关键词`

并明确：
- 空 query 不再走 unknown intent
- 失败路径已去除 “失败解释参考 / RAG失败解释测试” 污染文案

## 三、演示前准备

1. 启动 A 主系统
2. 启动 A 的飞书 listener
3. 启动 B 服务
4. 确认 B 运行在 127.0.0.1:8005
5. 确认 B 的以下接口可调用：
   - POST /internal/discovery/search
   - GET /internal/discovery/batches/{batch_id}
6. 确认 A → B HTTP 调用可用
7. 确认 Envelope 解包正常

## 四、演示步骤

### 第一步：在飞书里发送 discovery 搜索命令
例如：

搜索商品：wireless headphone

### 第二步：展示飞书返回
预期至少包含：

- 搜索关键词
- batch_id（若展示）
- 前 3~5 条候选结果
- 每条至少包含：
  - 序号
  - 名称
  - URL
  - 来源 / 站点（若有）

### 第三步：失败样本（可选）
若搜索为空或 discovery 失败，
飞书应返回老板可读错误文本，而不是异常堆栈。

## 五、演示成功标准

1. 飞书搜索命令能进入 A
2. A 能真实调到 B 的 discovery/search
3. A 能再调到 discovery/batches/{batch_id}
4. 飞书里有候选列表文本
5. 失败场景有老板可读文本
6. 不破坏 P10 / P11-A 已收口链路
