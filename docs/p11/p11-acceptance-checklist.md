# P11-C 验收清单

## 一、范围验收

- [x] 只做 add-from-candidates
- [x] 只依赖最近一次 discovery 结果
- [x] 未做卡片交互
- [x] 未做 candidate 多页翻页
- [x] 未做复杂状态机
- [x] 未做 pause / resume / delete
- [x] 未切 PostgreSQL
- [x] 未改 P10 / P11-A / P11-B 已收口边界

## 二、B 服务验收

- [x] B 运行在 127.0.0.1:8005
- [x] POST /internal/monitor/add-from-candidates 可调用
- [x] A 能访问 B
- [x] success / failed Envelope 均可处理

## 三、上下文验收

- [x] A 能保存最近一次 discovery 的 batch_id
- [x] A 能保存最小候选编号映射
- [x] 只对当前会话生效
- [x] 无上下文时能给出老板可读失败文本

## 四、飞书前台验收

- [x] 飞书里能先搜索出候选
- [x] 飞书里能发“加入监控第 N 个”
- [x] 成功时飞书能返回老板可读成功文本
- [x] 失败时飞书能返回老板可读失败文本
- [x] 不返回 Python 堆栈
- [x] 不返回原始 JSON

## 五、联动验收

- [x] add-from-candidates 成功后，B 中新增正式监控对象
- [x] 如继续执行“看看当前监控对象”，新增对象可见（若 B 返回语义允许）
- [x] 不破坏 P10 / P11-A / P11-B 查询与纳管链路

## 六、边界验收

- [x] A 仍是飞书入口层
- [x] B 仍是业务服务层
- [x] 未把 A / B 合并成一个项目
- [x] 未让 B 长期承担飞书入口角色
