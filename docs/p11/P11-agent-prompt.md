## 三、当前固定分工（必须继承）

A 负责：
- 接收飞书消息
- 解析老板意图
- 保存最小会话上下文
- 调用 B
- 把结果翻译成老板可读文本

B 负责：
- monitor targets
- pause / resume / delete
- 其它业务服务能力

不要把 A / B 合并成一个项目。

## 四、本轮唯一目标

只打通 3 个管理动作：

- POST /internal/monitor/{id}/pause
- POST /internal/monitor/{id}/resume
- DELETE /internal/monitor/{id}

形成最小闭环：

老板先发“看看当前监控对象”
→ A 返回对象列表
→ 老板再发“暂停监控第 N 个 / 恢复监控第 N 个 / 删除监控第 N 个”
→ A 从最近一次 targets 列表里取编号映射
→ A 调 B 的对应管理接口
→ A 返回成功或失败文本
→ 再看“当前监控对象”能看到状态变化
