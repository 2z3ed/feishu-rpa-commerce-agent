# P10 老板演示 SOP

## 一、演示目标

让老板在飞书里直接体验 A 接 B 的最小查询闭环。

固定演示链路：

老板发查询命令
→ A 解析命令
→ A 调 B 的 internal API
→ A 解包 Envelope
→ A 返回老板能看懂的飞书文本结果

## 二、演示固定样本

### 样本 1：今日摘要
飞书发送：
- 今天有什么变化

### 样本 2：当前监控对象
飞书发送：
- 看看当前监控对象

### 样本 3：商品详情（可选）
飞书发送：
- 看看商品 123 的详情

## 三、演示前准备

1. 启动 A 主系统
2. 启动 A 的飞书 listener
3. 确认 A 现有飞书文本收发正常
4. 启动 B 服务
5. 确认 B 运行在 127.0.0.1:8005
6. 确认以下接口可访问：
   - /internal/summary/today
   - /internal/monitor/targets
   - /internal/products/{id}/detail
7. 确认 A 调 B 不报连接错误
8. 确认 Envelope 可正常解包

## 四、演示步骤

### 第一步：今日摘要
在飞书中发送：
- 今天有什么变化

预期飞书返回：
- 今日摘要标题
- 今日重点变化
- 简短可读结论

### 第二步：当前监控对象
在飞书中发送：
- 看看当前监控对象

预期飞书返回：
- 当前监控对象列表
- 至少若干条对象摘要
- 明确是否为空

### 第三步：商品详情（可选）
在飞书中发送：
- 看看商品 123 的详情

预期飞书返回：
- 商品基本信息
- 关键状态
- 关键摘要

## 六、演示成功标准

1. 飞书命令能进入 A
2. A 能正确调用 B
3. A 能正确解包 Envelope
4. 飞书里有真实文本结果
5. 不需要老板打开 B 的后台
6. 不破坏 A 当前 P9 主线
